# 新人快速入口

如果你是第一次接手这个仓库，先不要把所有文档全看完。
按下面做就行。

## 我现在做什么，就先看哪个 skill

- 我做后端 / 接口 / 数据库：先看 `skills/cool-admin-midway-dev/SKILL.md`
- 我做后台 / 表格 / 表单 / CRUD：先看 `skills/cool-admin-vue-dev/SKILL.md`
- 我做 Uni / 移动端页面 / 路由：先看 `skills/cool-uni-dev/SKILL.md`
- 我做新功能、多人协作、要补文档：再看 `skills/cool-team-docs/SKILL.md`
- 我不知道任务怎么分级、跨端怎么走：看 `skills/cool-project-stack/SKILL.md`

## 小修复怎么走

适合这类事：改文案、样式、小 bug、小判断、小字段。

1. 先找对应端的 skill
2. 找一个最像的现有文件
3. 直接改
4. 改完跑 `docs/checklists/pre-commit.md`
5. 写清楚你改了什么

## 新功能怎么走

1. 先看 `skills/cool-project-stack/SKILL.md`
2. 先判断是 L2、L3 还是 L4
3. 再看对应端 skill
4. 再补文档 skill：需求、技术、实施
5. 涉及接口，补 API 文档
6. 涉及表结构，补数据库文档
7. 文档齐了再开发

## 跨端任务怎么走

默认顺序：

1. `cool-admin-midway`
2. `cool-uni`
3. `cool-admin-vue`

先把接口和数据结构说清楚，再分别开发，最后统一联调。

## 文档放哪里

统一放 `docs/`。

常见位置：

- 总入口：`docs/README.md`
- 系统长期说明：`docs/00-system/`
- 模块文档：`docs/01-modules/模块名/`
- 检查清单：`docs/checklists/`

## 你只想记住一句话

- 小修复：看端内 skill，直接改
- 新功能：先分级，再补文档，再开发
- 跨端：先后端，再 Uni，再后台

# 功能开发流程（超短版）

如果你接到一个功能，不知道先干嘛，就按这个来。

## 第 1 步：先判断这是什么任务

先判断 2 件事：

1. **这是哪一级？**
- L1：小修复、小改动
- L2：普通功能
- L3：复杂功能 / 多人协作
- L4：跨端 / 高影响功能

2. **这是哪一端？**
- 后端：接口、数据库、业务规则
- 后台：管理页面、表单、表格、CRUD
- uni：用户端页面、移动端流程
- 跨端：两个及以上端都要改

## 第 2 步：决定能不能直接写代码

- **L1**：一般可以直接改代码
- **L2**：建议先让 AI 写简版文档
- **L3 / L4**：先写文档，再开发

## 第 3 步：先找同类代码

不要凭空写。
先找：
- 当前项目里最像的功能
- 没有再看 base 模板

## 第 4 步：按顺序开发

### 单端功能
1. 看需求
2. 找同类代码
3. 改代码
4. 自测
5. 补改动说明

### 跨端功能
1. 先定接口和字段
2. 先做后端
3. 再做 uni / 后台
4. 最后联调
5. 补交付说明

## 第 5 步：开发完别直接结束

至少还要做：
- 自测
- 检查影响范围
- 补说明
- 跨端任务要联调

## 一句话记住

- 小修复：直接改，但先看同类代码
- 新功能：先让 AI 写文档，再开发
- 跨端：先后端，再前端，最后联调

## 不知道怎么跟 AI 说，就直接复制这段

```text
我接到一个功能，请先不要写代码，先帮我判断：
1. 这是 L1/L2/L3/L4 哪一级
2. 这是后端 / 后台 / uni / 跨端中的哪一种
3. 我应该先写文档还是直接改代码
4. 推荐开发顺序是什么
5. 代码应该优先看哪些同类实现
```

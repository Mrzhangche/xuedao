# 怎么跟 AI 说需求

这份文档给人直接看。

不会写提示词没关系，直接复制下面的模板，把你的需求替换进去就行。

---

## 1. 小修复怎么说

适合：
- 改文案
- 改样式
- 改一个小 bug
- 改一个小判断
- 改一个小字段显示或提交

直接复制：

```text
这是一个 L1 小改动，请直接按现有项目风格修改代码，不用先写长文档。

需求如下：
【把你的需求写在这里】

要求：
1. 先找同类实现
2. 只改必要范围，不要顺手重构
3. 改完告诉我改了哪些文件
4. 告诉我我应该怎么验证
```

---

## 2. 普通功能怎么说

适合：
- 新增一个普通页面
- 新增一组普通接口
- 给一个模块补完整功能

直接复制：

```text
这是一个 L2 功能，请先不要直接写代码，先按团队基座帮我整理开发内容。

需求如下：
【把你的需求写在这里】

请先输出：
1. 这是 L2 吗，为什么
2. 涉及后端 / 后台 / uni 中哪些端
3. 代码应该放哪些目录
4. 需要补哪些文档
5. 推荐开发顺序是什么

确认后，再先生成需求说明、技术设计、开发实施文档。
```

---

## 3. 复杂功能怎么说

适合：
- 新模块
- 新业务流程
- 多角色流程
- 多人协作任务

直接复制：

```text
这是一个 L3 功能，请先不要直接开发，先按团队基座整理完整开发方案。

需求如下：
【把你的需求写在这里】

请先输出：
1. 功能目标
2. 涉及角色
3. 涉及端
4. 页面 / 接口 / 数据库影响范围
5. 推荐开发顺序
6. 需要补哪些文档

然后先生成：
- 需求说明
- 产品设计
- 技术设计
- 开发实施
- 如果有接口变动，补 API 文档
- 如果有数据库变动，补数据库文档
```

---

## 4. 跨端功能怎么说

适合：
- 后端 + 后台
- 后端 + uni
- 后端 + 后台 + uni
- 核心流程改动

直接复制：

```text
这是一个 L4 跨端任务，请先不要直接改代码，先帮我拆开发顺序。

需求如下：
【把你的需求写在这里】

请先输出：
1. 涉及哪些端
2. 先做后端还是先做前端，为什么
3. 接口和字段应该先定什么
4. 各端分别改哪些内容
5. 联调时重点检查什么
6. 需要补哪些文档

确认清楚后，再按顺序推进开发。
```

---

## 5. 如果你现在连任务级别都判断不出来

直接复制：

```text
我接到一个需求，请先不要写代码，先帮我判断。

需求如下：
【把你的需求写在这里】

请你先告诉我：
1. 这是 L1 / L2 / L3 / L4 哪一级，为什么
2. 这是后端 / 后台 / uni / 跨端中的哪一种，为什么
3. 我应该直接改代码，还是先写文档
4. 推荐开发顺序是什么
5. 这类任务最容易漏什么
```

---

## 6. 如果你想让 AI 直接帮你拆步骤

直接复制：

```text
请把下面这个功能拆成“初级程序员可以直接照着做”的步骤，不要讲空话。

需求如下：
【把你的需求写在这里】

输出要求：
1. 每一步写清楚先做什么
2. 每一步写清楚去哪里改
3. 每一步写清楚改完怎么确认
4. 标明这是 L1/L2/L3/L4
5. 标明这是单端还是跨端
6. 如果需要先写文档，明确写出要产出哪些文档
```

---

## 7. 一句话记住

- 小修复：直接让 AI 改
- 普通功能：先让 AI 判断，再补简版文档
- 复杂功能：先让 AI 出方案，再开发
- 跨端功能：先拆顺序，再分别开发

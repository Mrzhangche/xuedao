# 文档导航 README

## 这份文档的作用

这份文档是给人看的总入口，帮助阅读者先快速理解系统全貌，再顺着目录继续往下看。

它不负责写模块细节，只负责回答：

- 这个系统是做什么的
- 有哪些端和角色
- 核心模块有哪些
- 文档分别放在哪里
- 推荐先看哪几份文档

## 文档目录说明

### `docs/00-system`

- `system-overview.md`
  用来看系统整体结构、端划分、模块边界、权限角色、部署依赖
- `system-function-manual.md`
  用来看系统已经交付的功能入口、使用方式、限制条件

### `docs/01-modules`

每个模块一个目录，里面按固定顺序维护需求、产品、技术、数据库、接口、实施、测试、交付文档。

## 文档维护约定

- 新模块开始前，先建立模块目录和前期文档
- 功能完成后，至少更新测试、交付、系统功能说明
- 如果影响系统结构、角色权限、入口路径或外部依赖，同步更新 `system-overview.md`
- 如果新增了模块或阅读入口变化，同步更新本文档
